## 三、新发现的问题（第三次评估）

### 3.1 CrawlerControlPanel 中投注确认对话框引用未定义函数 🔴 严重

| 项目 | 内容 |
|------|------|
| **问题描述** | [CrawlerControlPanel.tsx L445-L472](file:///d:/下载/足球竞彩量化分析系统/足球竞彩量化分析系统/src/components/corner/CrawlerControlPanel.tsx#L445-L472) 的投注确认对话框中，按钮引用 `cancelBetConfirm`（第 458 行）和 `confirmExecuteBets`（第 464 行），但这两个函数**未在组件中定义** |
| **影响分析** | `showBetConfirm` 状态初始化为 `false` 且无代码将其设为 `true`，因此该对话框目前不会渲染，未定义函数不会被调用。但这是一段**悬挂的死代码**，未来任何人将 `showBetConfirm` 设为 `true` 后将立即引发 **ReferenceError**。即使作为占位代码也缺乏防错处理 |
| **风险等级** | 🔴 **严重**（触发即崩溃） |
| **改进建议** | ① 定义 `cancelBetConfirm` 和 `confirmExecuteBets` 两个函数；② 或将对话框与 `handleExecutePendingBets`（第 288 行）连接，使对话框拦截生效 |

### 3.2 前端自动刷新间隔与后端轮询间隔仍然不匹配 🟡 中

| 项目 | 内容 |
|------|------|
| **问题描述** | [CrawlerControlPanel.tsx L328](file:///d:/下载/足球竞彩量化分析系统/足球竞彩量化分析系统/src/components/corner/CrawlerControlPanel.tsx#L328) 前端自动刷新固定 10s，后端 POLL_INTERVAL 为 5000ms（5s）。前端 10s 间隔意味着每次刷新拿到的都是 5-10 秒前的缓存数据 |
| **影响分析** | 用户看到的比赛数据延迟 5-10 秒，对需要快速反应的策略（如策略一 35'-55' 窗口）可能错过窗口期 |
| **风险等级** | 🟡 中 |
| **改进建议** | 将前端自动刷新间隔调整为与后端一致（5000ms），或读取后端 `POLL_INTERVAL` 动态设置 |
