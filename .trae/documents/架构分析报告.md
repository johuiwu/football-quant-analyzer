## 二、架构设计分析

### 2.1 整体架构评估

#### 优点
1. **前后端分离架构清晰** - 明确的职责划分
2. **多技术栈整合合理** - 前端React、后端Node.js、爬虫Python各司其职
3. **模块化设计** - 核心业务逻辑相对独立
4. **支持桌面应用** - Electron打包便于部署

#### 问题与不足
1. **架构耦合度偏高** - 前端业务逻辑与UI组件耦合紧密
2. **状态管理混乱** - 同时使用Context、useState、useRef管理状态，缺乏统一规划
3. **后端与前端模型重复** - Poisson预测在前后端都有实现，但逻辑不一致
4. **缺乏架构文档** - 没有清晰的架构设计文档和数据流说明

### 2.4 数据计算逻辑分析

#### 2.4.1 核心算法实现

##### 1. Poisson分布计算
**文件**: `src/utils/quantModel.ts` (前端)、`backend/services/poissonPredictor.js` (后端)

**实现逻辑**:
```typescript
export function poisson(k: number, lambda: number): number {
  if (lambda <= 0) return k === 0 ? 1 : 0;
  const exp = Math.exp(-lambda);
  let factorial = 1;
  for (let i = 1; i <= k; i++) {
    factorial *= i;
  }
  return (Math.pow(lambda, k) * exp) / factorial;
}
```

**评估**:
- 优点: 实现简洁清晰
- 问题: 
  - 前后端实现不一致（前端用阶乘，后端用对数阶乘）
  - 缺少大数优化（k>20时阶乘会溢出）
  - 没有数值稳定性保障

##### 2. Dixon-Coles调整
**文件**: `src/utils/quantModel.ts`

**实现逻辑**:
```typescript
export function dixonColesAdjustment(
  x: number, y: number, lambda: number, mu: number, rho: number = -0.075
): number {
  if (x === 0 && y === 0) return 1 - lambda * mu * rho;
  if (x === 1 && y === 0) return 1 + mu * rho;
  if (x === 0 && y === 1) return 1 + lambda * rho;
  if (x === 1 && y === 1) return 1 - rho;
  return 1.0;
}
```

**评估**:
- 优点: 正确实现了Dixon-Coles低比分调整
- 问题: 
  - rho参数硬编码为-0.075，无法根据联赛调整
  - 缺少理论依据说明
  - 没有参数敏感性分析

##### 4. 凯利公式
**文件**: `src/utils/quantModel.ts`

**实现逻辑**:
```typescript
export function calculateKellyFraction(prob: number, decimalOdds: number): number {
  if (decimalOdds <= 1.0 || prob <= 0) return 0;
  const b = decimalOdds - 1.0;
  const q = 1.0 - prob;
  const f = (prob * b - q) / b;
  const halfKelly = f * 0.5;  // 使用半凯利
  return Math.max(0, parseFloat((halfKelly * 100).toFixed(2)));
}
```

**评估**:
- 优点: 正确实现了凯利公式，使用半凯利更稳健
- 问题: 
  - 缺少对凯利值的阈值限制
  - 没有考虑赔率波动风险
  - 缺少凯利公式的历史回测验证

### 4.2 数据计算逻辑优化

#### 1. Poisson分布优化
```typescript
// 使用对数阶乘提高数值稳定性
export function poisson(k: number, lambda: number): number {
  if (lambda <= 0) return k === 0 ? 1 : 0;
  if (k > 170) return 0;  // 防止溢出
  const logFact = logFactorial(k);
  return Math.exp(-lambda + k * Math.log(lambda) - logFact);
}

function logFactorial(n: number): number {
  if (n <= 1) return 0;
  let result = 0;
  for (let i = 2; i <= n; i++) {
    result += Math.log(i);
  }
  return result;
}
```

#### 2. Elo动态更新机制
```typescript
interface EloUpdate {
  teamId: string;
  opponentElo: number;
  result: 'win' | 'draw' | 'loss';
  kFactor: number;
}

export function updateElo(currentElo: number, update: EloUpdate): number {
  const expected = 1 / (1 + Math.pow(10, (update.opponentElo - currentElo) / 400));
  let actual: number;
  switch (update.result) {
    case 'win': actual = 1; break;
    case 'draw': actual = 0.5; break;
    case 'loss': actual = 0; break;
  }
  return Math.round(currentElo + update.kFactor * (actual - expected));
}
```
