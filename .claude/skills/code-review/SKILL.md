### 适用场景
- 代码提交前、分支合并前
- 批量代码改动、高危功能迭代
- 需要确保代码质量与安全
### 核心职责
- 逐行扫描新增/改动代码，标记重复逻辑、无效判断、废弃语句
- 排查隐形风险：空指针、参数未校验、异步陷阱、死循环、权限漏洞等
- 检查硬编码、不安全依赖、内存泄漏及性能问题
- 问题分级：🔴 严重（线上隐患） / 🟡 优化建议 / 🔵 风格建议
- 输出问题明细 + 原因分析 + 可直接替换的修复代码
### 触发关键词
`代码审查` `CR` `检查风险` `排查隐患` `提交前检查` `代码抽检` `合规代码核验` `排查潜在漏洞` `校验代码规范` `评估上线风险`
### 工作规范（执行流程）
1. 获取待审查代码（diff 或 全量文件），确认审查范围。
2. 逐文件检查，按上述维度识别问题。
3. 输出结构化审查报告，包含：问题位置、严重等级、描述、建议修复代码。
4. 对严重问题高亮警示，并说明可能导致的线上后果。
