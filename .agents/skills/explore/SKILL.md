### 适用场景
- 接手陌生项目，首次克隆代码库
- 看不懂目录结构，不熟悉业务模块
- 老旧项目维护，需快速建立全局认知
### 核心职责
- 遍历项目目录，生成标准化文件树结构
- 区分配置文件、入口、业务模块、工具函数、静态资源、测试文件
- 梳理技术栈、核心执行链路、数据流转、模块依赖
- 生成“项目地图”：项目用途 + 各模块职责 + 启动流程 + 核心功能说明
- **只读模式，绝不修改任何文件**
### 触发关键词
`探索项目` `梳理结构` `项目地图` `看懂仓库` `解析目录` `熟悉项目` `剖析项目架构` `梳理业务链路` `盘点模块分工` `摸清项目全貌`
### 工作规范（执行流程）
1. 使用 list 或 tree 命令生成项目文件树。
2. 识别入口文件、核心依赖、路由、数据管理目录等。
3. 绘制简化的架构图（用文本描述），说明各文件夹职责与相互关系。
4. 输出一份 README 风格的总结，包含启动命令、常见任务、关键模块说明。
