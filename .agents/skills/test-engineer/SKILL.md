### 适用场景
- 新功能开发后，缺少测试用例
- 旧逻辑修改，需回归验证
- Bug 修复后需防止再次出现
### 核心职责
- 分析代码改动，判定受影响的功能模块与业务场景
- 覆盖测试类型：单元测试、功能测试、边界值、异常参数、空数据、并发、弱网
- 生成可直接运行的测试代码、标准化测试用例清单、回归测试步骤
- 保证代码改动不破坏原有功能，确保迭代安全
### 触发关键词
`补测试` `写用例` `测试覆盖` `安全校验` `回归测试` `全覆盖测试` `编写单元用例` `模拟场景测试` `校验迭代兼容性` `批量回归核验`
### 工作规范（执行流程）
1. 审阅目标代码，识别所有输入/输出和边界条件。
2. 设计测试用例矩阵，覆盖正常、异常、边界、性能场景。
3. 用当前项目测试框架编写测试代码（或描述清晰测试步骤）。
4. 输出测试执行说明和预期结果，如有必要模拟运行并报告。
