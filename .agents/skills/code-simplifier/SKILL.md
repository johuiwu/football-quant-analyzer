### 适用场景
- 函数体庞大、嵌套过深（超过3层if/for）
- 逻辑繁琐臃肿，维护成本高
- 祖传代码，需提升可读性和可维护性
### 核心职责
- **100% 保留原有业务功能，零逻辑变更，零功能丢失**
- 扁平化多层嵌套，拆解超长函数（建议不超过 50 行）
- 简化复杂条件判断（如卫语句、查表法替代if-else）
- 合并重复分支，抽离公共逻辑为独立函数
- 重构后给出前后对比，确保逻辑等价性
### 触发关键词
`精简代码` `重构优化` `简化逻辑` `梳理臃肿代码` `扁平化代码` `拆解庞杂函数` `简化条件分支` `优化代码架构` `提纯核心逻辑`
### 工作规范（执行流程）
1. 识别需要重构的目标代码块，向用户确认重构范围。
2. 分析输入输出，设计保留逻辑的重构方案。
3. 执行重构，输出新代码，并附“逻辑等价性说明”。
4. 建议运行现有测试（或第10技能）来验证功能无损。
