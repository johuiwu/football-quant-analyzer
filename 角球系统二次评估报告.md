## 二、新发现的问题

以下为本次二次评估中新发现的 10 项问题：

### 新问题 1：cornerRoutes.js 语法错误导致路由注册异常 🔴 严重

| 项目 | 内容 |
|------|------|
| **问题描述** | [cornerRoutes.js 第 47 行](file:///d:/下载/足球竞彩量化分析系统/足球竞彩量化分析系统/backend/routes/cornerRoutes.js#L47)的 `});` 提前关闭了 `GET /corner/strategies/check` 的 try 块，导致紧随其后的 `PUT /corner/strategies` 路由（第 48 行）被错误地包裹在 try...catch 结构之外 |
| **影响分析** | 该文件可通过 Node.js 语法检查，但 Express 路由注册顺序可能出现异常。`PUT /corner/strategies` 路由可能被错误注册或无法正常工作，导致前端策略同步功能（`syncStrategiesToBackend`）失效 |
| **风险等级** | 🔴 **严重** |
| **改进建议** | 将第 47 行的 `});` 移至第 63 行的 `});` 处（即 `PUT` 路由的 `catch` 块之后），确保两个路由各自有完整的 try-catch 包裹 |

### 新问题 2：SettingsPanel 余额获取路径错误 🟡 中

| 项目 | 内容 |
|------|------|

### 新问题 3：后端轮询间隔默认值不一致 🟡 中

| 项目 | 内容 |
|------|------|
| **问题描述** | [cornerCrawler.js 第 20 行](file:///d:/下载/足球竞彩量化分析系统/足球竞彩量化分析系统/backend/services/cornerCrawler.js#L20) 默认值 10000ms，[cornerService.js 第 11 行](file:///d:/下载/足球竞彩量化分析系统/足球竞彩量化分析系统/backend/services/cornerService.js#L11) 默认值 5000ms，两者读取相同的环境变量 `CRAWLER_POLL_INTERVAL` |

### 新问题 4：前端策略评估逻辑与后端存在路径差异 🟡 中

| 项目 | 内容 |
|------|------|
| **问题描述** | `cornerStore.ts` 的 `evaluateMatchForStrategies()` 中，条件判断顺序与 `cornerEvaluator.js` 不完全一致。后端先检查 `leadGoals >= 20`（哨兵值），再检查 `leadGoals > 0 && leadGoalsWeak === 0`；前端则是先检查 `leadGoals === 0 && leadGoalsWeak === 0`，再检查 `leadGoals >= 20`。虽然当前策略参数（leadGoals=0,1,2,3,99）下结果一致，但逻辑路径不同 |
| **影响分析** | 未来修改策略参数时，前后端可能产生不一致的评估结果 |
| **风险等级** | 🟡 中 |
| **改进建议** | 重构前端评估逻辑，使其与 cornerEvaluator.js 的条件判断顺序完全一致；或移除前端独立实现，全部通过 API 获取评估结果 |
